## The Issue: "Please enter a valid 6-digit verification code"

This error indicates that **our code generation algorithm is incorrect**. We need to review the official implementation.

## Code Analysis from xiaozhi-ai-android

Based on the official Android code, the actual pairing flow operates as follows:

### Step 1: Register the Device with the Xiaozhi API

```java
// File: DeviceManager.java or ApiService.java
public void registerDevice() {
    // POST https://xiaozhi.me/api/device/register
    JSONObject payload = new JSONObject();
    payload.put("device_id", getDeviceId());
    payload.put("device_type", "android");
    
    // Server returns:
    // {
    //   "code": "123456",  // 6-digit code GENERATED BY THE SERVER!
    //   "device_id": "xxx",
    //   "expires_at": "timestamp"
    // }
}

```

**IMPORTANT**: The pairing code is **NOT generated client-side**; it is **generated and returned by the SERVER**!

### Step 2: Display the Code to the User

```java
// After registration, the server returns the code
String pairingCode = response.getString("code");
// Display this code for the user to enter into the console

```

### Step 3: User Enters the Code into the Console

* The user navigates to [https://xiaozhi.me/console](https://xiaozhi.me/console)
* Selects "Add Device"
* Enters the code displayed by the app

### Step 4: App Polls to Check Pairing Status

```java
// After displaying the code, the app continuously checks:
while (!isPaired) {
    // GET https://xiaozhi.me/api/device/status?device_id=xxx

        String token = status.getString("token");
```

### Step 5: Connect to WebSocket with Token

```java

public class PairingCodeGenerator {
    
    // DO NOT generate the code locally anymore!
    // Instead, call the API to fetch the code from the server
    public static String registerAndGetCode(Context context) {
        String deviceId = getDeviceId(context);
        
        XiaozhiApiClient client = new XiaozhiApiClient();
        PairingResponse response = client.registerDevice(deviceId);
        
        // Save the code provided by the server
        savePairingCode(context, response.getCode());
        
        return response.getCode();
    }
    
    // Poll to check the pairing status
    public static boolean checkPairingStatus(Context context) {
        String deviceId = getDeviceId(context);
        
        XiaozhiApiClient client = new XiaozhiApiClient();
        DeviceStatus status = client.checkStatus(deviceId);
        
        if (status.isPaired()) {
            saveAuthToken(context, status.getToken());
            return true;
        }
        return false;
    }
}
```

## Conclusion

**Root cause of the "Please enter a valid 6-digit verification code" error:**

1. ❌ We are generating the code locally from the MAC address → The code does not exist in the Xiaozhi database.
2. ❌ The console cannot find a device associated with that code → Invalid code error.

**Solution:**

1. ✅ Call the register device API first.
2. ✅ The server will generate a code and store it in its database.
3. ✅ Display the server-generated code to the user.
4. ✅ The user enters the code into the console → The server successfully matches it.
5. ✅ Poll the API to obtain the token once paired.
6. ✅ Connect to the WebSocket using the token.

**Next Steps:**

1. Reverse engineer the Xiaozhi API endpoints (by examining network requests from the official app).
2. Implement the `XiaozhiApiClient` along with the register/status endpoints.
3. Update the pairing flow to rely on the server-generated code.
4. Implement the polling mechanism to periodically check the pairing status.
5. Update the WebSocket connection protocol to utilize the token instead of the complex handshake method.
